📡 Robin Post-Market Report — 数据调用流程

OpenClaw Cron → API Server → Robin → Discord

⏰ 1. OpenClaw Cron 触发

每个交易日 1:30 PM PT，OpenClaw 的 cron 系统唤醒 Robin agent。
Cron 把预写好的 prompt（含持仓列表 + API调用指令）发给 Robin。
cron: 30 13 * * 1-5 agent: robin (Sonnet)

📊 2. Robin 调用 /holdings/rank

Robin 用 web_fetch 工具请求这个端点。
服务器在这一步做所有计算：读取 scan_results.json → 排序 791 只股票 → 只返回 8 只持仓的排名。
GET /holdings/rank ~200 bytes 服务器排序
// 服务器返回 — 预计算好的，Robin 直接用 { "holdings_ranked": [ {"ticker": "CF", "rank": 2, "composite_score": 89.55}, {"ticker": "CTRA", "rank": 3, "composite_score": 88.28}, {"ticker": "EQT", "rank": 25, "composite_score": 78.97}, // ... 总共 8 只 ], "total_stocks": 791 }

🛡️ 3. Robin 调用 /risk/summary

获取风险数据：每只持仓的当日 P&L、trailing stop 警报、停火预警、仓位限制。
这个端点也是预计算的 — 服务器用 yfinance 抓实时价格，算好 P&L 后返回。
GET /risk/summary ~2 KB
// 包含：positions (P&L), trailing_stops, ceasefire, oil_monitor { "positions": [{"ticker": "CF", "pnl": +110.48, "pnl_pct": 25.08}...], "ceasefire": {"urgency": "HIGH", "signals": [...]}, "trailing_stops": {"alerts": [{"ticker": "CF", "status": "TRIGGERED"}]} }

⚖️ 4. Robin 调用 /rebalance/status

检查是否有持仓需要调仓（信号变化、持有天数到期、新的 Top 20 推荐）。
GET /rebalance/status ~500 bytes

✍️ 5. Robin 整理 + 写报告

Robin 把 3 个 API 的数据组合成人类可读的报告。
这是 LLM 真正擅长的事：把结构化数据变成自然语言。
所有数字直接从 API 响应复制，不需要计算。
Sonnet 强项：格式化 + 写作

📢 6. 发送到 #stock-picker

Robin 用 message 工具把报告发到 Discord #stock-picker 频道。
如果检测到停火信号，同时发送警报到 #red-note。
#stock-picker 如有停火 → #red-note
