Docker 是什么？

Docker 把你的 代码 + 依赖 + 运行环境 打包成一个"盒子"（Container）。

这个盒子拿到 任何电脑 上都能跑，不用重新装任何东西。

三个核心概念

📝 Dockerfile

像菜谱一样的文本文件。告诉Docker：用什么基础环境、装什么依赖、复制什么文件、怎么启动。

类比：宜家家具的安装说明书

📦 Image

按照Dockerfile构建出来的"成品"。只读的，像一个模板。可以分享给别人。

类比：装好的宜家家具（还没搬进房间）

🚀 Container

Image跑起来后就是Container。可以同时跑多个Container（从同一个Image）。

类比：家具搬进房间，开始使用了

没Docker vs 有Docker

❌ 没有 Docker

1. 在Mac上开发，跑得好好的 ✅

2. 部署到Linux服务器...

3. Python版本不对 ❌

4. 装了一堆依赖还是报错 ❌

5. 数据库连不上 ❌

6. "在我电脑上明明能跑啊！" 😤

✅ 有 Docker

1. 写好Dockerfile 📝

2. docker build 打包 📦

3. 推到任何服务器

4. docker run 一键启动 🚀

5. 完全一样的环境 ✅

6. "在哪都能跑" 😎

Stock Picker 的例子

❌ 现在的方式

# 新电脑上要做的事：

$ brew install python@3.11

$ pip install fastapi uvicorn yfinance pandas xgboost lightgbm scipy scikit-learn ...

ERROR: lightgbm wheel build failed

$ brew install libomp

$ pip install lightgbm

$ git clone ... && cd stock-picker

$ cp .env.example .env && vim .env

$ python -m uvicorn src.api:app

# 30分钟后终于跑起来了...

✅ Docker 方式

# 新电脑上只需要：

$ docker run -p 8000:8000 stock-picker

INFO: Started server on 0.0.0.0:8000

# 完了。就这一行。

 

 

 

 

 

# 10秒搞定 ✅

📄 Stock Picker 的 Dockerfile（逐行解释）

FROM python:3.11-slim ← 基础镜像：用Python 3.11（精简版，没有多余的东西）
WORKDIR /app ← 在容器里创建 /app 文件夹，后面的操作都在这里
COPY requirements.txt . ← 先只复制依赖列表（利用Docker缓存，依赖没变就不重装）
RUN pip install -r requirements.txt ← 安装所有Python依赖
COPY src/ src/ ← 复制源代码
COPY static/ static/ ← 复制前端文件（dashboard）
COPY config.yaml . ← 复制配置文件
COPY data/ data/ ← 复制数据文件（scan结果、holdings等）
EXPOSE 8000 ← 告诉Docker这个容器用8000端口
CMD ["uvicorn", "src.api:app", "--host", "0.0.0.0", "--port", "8000"] ← 容器启动时运行的命令

💻 Docker 常用命令

docker build -t name . 构建Image（-t 起名字，. 是当前目录）
docker run -p 8000:8000 name 运行Container（-p 映射端口）
docker ps 查看运行中的Container
docker stop [id] 停止Container
docker images 查看所有Image
docker logs [id] 查看Container日志
docker exec -it [id] bash 进入Container内部（像SSH）
docker push user/name 推到Docker Hub（给别人用）

面试常问的Docker问题

Q: Docker和虚拟机有什么区别？

Docker共享宿主机内核，轻量快速（秒级启动）。VM有独立OS，重量级（分钟级启动）。Docker适合微服务，VM适合需要完全隔离的场景。

Q: Image和Container的区别？

Image是只读的模板（像类class），Container是Image跑起来的实例（像对象instance）。一个Image可以启动多个Container。

Q: 为什么先COPY requirements.txt再COPY代码？

利用Docker layer缓存。依赖不常变，代码常变。这样改代码不用重新安装依赖，build更快。

Q: -p 8000:8000 是什么意思？

左边是宿主机端口，右边是Container内部端口。把Container的8000端口映射到你电脑的8000端口。

🧠 测测你学到了什么

1. Docker的核心价值是什么？

A. 让程序跑得更快
B. 保证代码在任何环境都能一样地运行
C. 替代GitHub

2. Dockerfile是什么？

A. 描述如何构建Docker Image的说明文件
B. Docker的配置文件，存放密码
C. Docker的日志文件

3. 为什么Dockerfile里先COPY requirements.txt，再COPY代码？

A. 因为requirements.txt更重要
B. 没有特别原因，顺序无所谓
C. 利用Docker缓存 — 依赖没变就不用重新安装，build更快

4. docker run -p 8000:8000 的 -p 是什么意思？

A. 设置密码 (password)
B. 端口映射 (port mapping) — 宿主机:容器
C. 设置进程数 (processes)
